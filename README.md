# Team Builder iOS

> SwiftUI-приложение для сборки сильных команд: с профилями сотрудников, DISC и мотивационными опросами, pulse-чеками, уведомлениями и менеджерской аналитикой по рискам.

## Что Это

**Team Builder** помогает командам видеть не только роли и должности, но и динамику взаимодействия:

- сотрудник видит свой профиль, результаты опросов и pulse-состояние;
- руководитель получает обзор команды, карту ролей и сигналы риска;
- приложение умеет работать в demo-режиме и переживает временную потерю сети через офлайн-очередь.

Интерфейс построен на `SwiftUI`, состояние собрано в `AppModel`, а данные приходят либо из live API, либо из встроенного mock-клиента.

## Что Внутри

### Сценарии сотрудника

- домашний экран с персональной сводкой;
- DISC-оценка;
- мотивационный опрос;
- weekly pulse-check;
- центр уведомлений;
- редактирование профиля.

### Сценарии руководителя

- обзор команды;
- role map и карточки сотрудников;
- экран рисков и управленческих сигналов;
- уведомления с deeplink-навигацией;
- доступ к профилям и кратким выводам по людям.

### Технические особенности

- `SwiftUI` + `@MainActor` state management;
- `async/await` для сетевого слоя;
- `Keychain` для токенов;
- офлайн-очередь отправок с повторной синхронизацией;
- поддержка deep links;
- кастомная типографика (`HalvarBreitt`, `T2 Rooftop`);
- live API и отдельный `MockAPIClient` для demo-flow и превью.

## Архитектура

`AppModel` выступает как единая точка координации:

- хранит сессию и выбранную роль;
- подгружает employee/manager dashboard;
- переключает источник данных между live и demo;
- управляет алертами, уведомлениями и состоянием синхронизации;
- ставит отправки в очередь при отсутствии сети и синхронизирует их позже.

Это делает проект удобным для быстрых продуктовых итераций: UI уже отделен от источника данных, а demo-flow помогает показывать сценарии даже без стабильного backend-окружения.
